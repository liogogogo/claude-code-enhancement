# Code Review Skill

对代码进行全面的审查，发现问题并提出改进建议。

## 审查流程

1. **获取变更文件**
   ```bash
   git diff --name-only HEAD~1
   ```
   或审查用户指定的文件。

2. **审查维度**

   ### 安全性 🔒
   - SQL 注入风险
   - XSS 跨站脚本
   - 敏感信息泄露（API key、密码）
   - 不安全的依赖

   ### 代码质量 📝
   - 命名规范
   - 重复代码
   - 复杂度过高
   - 未使用的代码

   ### 性能 ⚡
   - N+1 查询
   - 不必要的循环
   - 内存泄漏风险
   - 低效算法

   ### 最佳实践 ✅
   - 错误处理
   - 日志记录
   - 类型注解
   - 文档注释

3. **输出格式**

   ```markdown
   ## Review 结果

   ### 🔴 必须修复
   - [文件:行号] 问题描述

   ### 🟡 建议改进
   - [文件:行号] 问题描述

   ### ✅ 做得好
   - 值得肯定的代码

   ## 总结
   总体评价和改进优先级建议
   ```

## 注意事项

- 专注于实际问题，避免过度建议
- 给出具体的修复代码示例
- 按严重程度排序问题
